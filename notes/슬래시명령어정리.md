# Claude Code 슬래시 명령어 정리

> 입력창에 `/`를 치면 뜨는 특수 명령어 모음. 자연어 대화와 별개로 `/`로 시작하는 게 특징.
> 아래는 주요 명령어만 정리한 것이며, 실제로는 더 많음 (`/` 입력 후 스크롤).

---

## ⭐ 당장 기억할 것 (초보자용)

처음부터 다 외울 필요 없음. 우선 이 세 개만:

| 명령어 | 용도 |
| --- | --- |
| `/clear` | 작업이 바뀔 때 대화 비우기 |
| `/codex:review` | 코드 리뷰 (Codex 플러그인) |
| `/exit` | Claude Code 종료 |

나머지는 필요할 때 `/` 쳐서 찾으면 됨.

---

## 프로젝트 설정 관련

| 명령어 | 설명 |
| --- | --- |
| `/init` | 폴더를 분석해서 `CLAUDE.md`를 만듦 |
| `/add-dir` | 작업 폴더를 추가로 더 붙임 (여러 폴더 동시 작업 시) |
| `/config` | 설정 패널을 엶 |
| `/doctor` | Claude Code 설치·설정에 문제가 없는지 진단 (뭔가 이상할 때) |

---

## 대화·컨텍스트 관리 (실무에서 가장 자주 씀)

| 명령어 | 설명 |
| --- | --- |
| `/clear` | 대화를 비우고 새로 시작. 이전 세션은 디스크에 남아 `/resume`으로 복구 가능 |
| `/compact` | 지금까지의 대화를 요약해 컨텍스트 공간 확보. 대화가 길어져 느려질 때 |
| `/context` | 현재 컨텍스트 사용량을 색깔 그리드로 표시 |
| `/copy` | Claude의 마지막 응답을 클립보드에 복사 |

> **`/compact` vs `/clear`** — compact는 요약해서 *이어가고*, clear는 *통째로 비움*.

## Codex 플러그인

| 명령어 | 설명 |
| --- | --- |
| `/codex:setup` | Codex CLI 준비 상태 확인, review gate 켜고 끄기 |
| `/codex:review` | 현재 작업한 코드를 Codex가 리뷰 |
| `/codex:adversarial-review` | 설계 방향까지 따지는 비판적 리뷰 |
| `/codex:rescue` | 버그 조사·수정을 Codex에 위임 |
| `/codex:status` | 백그라운드 Codex 작업 진행 상황 확인 |
| `/codex:result` | 완료된 Codex 작업 결과 확인 |
| `/codex:cancel` | 진행 중인 백그라운드 Codex 작업 취소 |

## 기타

| 명령어 | 설명 |
| --- | --- |
| `/agents` | 에이전트 설정 관리 |
| `/advisor` | 중요한 순간에 더 강한 모델에게 조언을 구하도록 설정 |
| `/autofix-pr` | PR(풀 리퀘스트) 이슈를 모니터링하고 자동 수정 (git/GitHub 워크플로우용) |
| `/background` | 현재 세션을 백그라운드로 돌리고 터미널을 비움 |
| `/branch` | 지금 대화 시점에서 분기 생성 ("여기서부터 다르게 시도" 할 때) |
| `/btw` | 메인 대화를 끊지 않고 곁다리 질문 ("by the way") |
| `/chrome` | Claude in Chrome (베타) 설정 |
| `/color` | 이번 세션의 프롬프트 바 색상 변경 |
| `/desktop` | 현재 세션을 Claude 데스크탑 앱에서 이어감 |
| `/diff` | 아직 커밋 안 된 변경 사항을 봄 |
| `/effort` | 모델이 얼마나 "공들여" 작업할지 수준 설정 |
| `/resume` | 이전 대화 목록에서 골라 이어가기 |
| `/exit` | Claude Code 종료 |

---

## 사용 팁

- `/`만 친 상태에서 **위아래 화살표**로 항목 선택 가능
- `/cle`처럼 **일부만 쳐서** 후보를 좁힐 수 있음
- 목록을 닫고 자연어 입력으로 돌아가려면 **Esc** 또는 **Backspace**로 `/` 삭제
